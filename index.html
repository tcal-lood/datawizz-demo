How Datawizz works
Under the hood — invisible to engineers, transformative for the invoice
01
Classify task complexity
Every incoming query is analyzed for task complexity — pattern-match vs. reasoning required. This decision happens in milliseconds before the query touches any LLM.
02
Route to the right model
Low-complexity queries go to a specialized 3B model. Medium complexity to a 7B model. Only genuinely hard queries reach the frontier model. Engineers never see the difference.
03
Learn and improve
The platform learns from every routing decision. Models improve continuously in production through reinforcement learning — the system gets smarter and cheaper over time.
